# 2026-04-07 Daily Memory

## 占卜网站修复 (Bug Fixes)

### 1. Hydration 错误 (Next.js SSR)
- **问题**：Next.js `.next` 缓存损坏，导致服务端 HTML 和客户端 hydration 不匹配，报错 "Text content does not match server-rendered HTML"
- **原因**：监控脚本反复重启 `next dev` 失败（`npm` 命令路径问题），损坏缓存
- **修复**：删除 `.next` 缓存目录，重启前端服务
- **预防**：修复监控脚本的 `npm` 路径问题（见下方）

### 2. `guaxiang` 字段缺失
- **问题**：前端 `page.tsx` 调用 `guaxiang.name` 时报错 undefined
- **原因**：后端 `compute_hexagram_from_throws()` 返回值缺少 `guaxiang` 字段，且 Pydantic `DivinationResponse` 模型也未声明该字段
- **修复**：
  1. 在 `compute_hexagram_from_throws()` 返回字典中添加 `guaxiang`
  2. 在 `DivinationResponse` Pydantic 模型中添加 `guaxiang: Optional[Dict[str, Any]] = None`

### 3. 京房八宫卦查找失败（动爻 ≥3 时体用五行错误）
- **问题**：3 动爻时，AI 把履卦（五行土）错误定为金
- **原因**：`_find_gua_gong_and_wuxing()` 只能匹配完整卦名（如"天泽履"），无法匹配简称（如"履"），导致 fallback 到错误的八卦五行
- **修复**：将卦名查找改为支持部分匹配（包含判断），优先精确匹配完整卦名，再匹配简称

### 4. 前端端口漂移（3000 vs 10008）
- **问题**：前端有时跑在 3000 端口而非规定的 10008
- **原因**：监控脚本启动失败后，前端手动重启时端口被 Next.js 动态分配
- **修复**：杀死旧进程后，明确指定 `PORT=10008 npm run dev`

### 5. 监控脚本 `npm` 路径问题
- **问题**：监控脚本中 `npm` 命令找不到，导致前端无法自动重启
- **原因**：`which npm` 可能返回空或路径不对
- **修复方案**：使用 `/root/.nvm/versions/node/v22.22.1/bin/npm` 或在脚本中先确认 npm 路径
